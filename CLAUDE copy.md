# CLAUDE.md — Nixon Jach Hubbard Law Firm Website

## Project Overview
A complete website rebuild for Nixon Jach Hubbard, PLLC — a Dallas-based law firm specializing in construction law, commercial real estate, business litigation, and corporate transactions. The current site is a dated Wix template with poor SEO, no content strategy, and zero lead generation.

## Tech Stack
- Next.js 14+ (App Router)
- TypeScript
- Tailwind CSS
- Framer Motion (animations)
- Deployed to Vercel

## Design Philosophy
This site must look **hand-crafted by a human designer**, not generated by AI. No generic layouts. No symmetrical 3-column grids. No stock-photo-with-overlay heroes. The firm's brand is **authoritative, experienced, and quietly confident** — think high-end architecture firm website, not a template law firm site.

### Design Rules
- Use a serif typeface for headlines (suggests tradition, authority)
- Use a clean sans-serif for body text (suggests clarity, modernity)
- Asymmetric layouts — offset images, text that breaks the grid
- Generous whitespace — let content breathe
- Thin rule lines as design accents
- Dark navy / charcoal palette with warm gold accents
- All images must be real photography of the firm, Dallas skyline, or construction projects — never generic stock
- Subtle scroll-triggered animations (20-30px translate, 0.4s ease)

## COPY SYSTEM — READ BEFORE WRITING ANY TEXT

**Before writing ANY copy for this site, read `docs/COPY_SYSTEM.md` in full.** That file contains the complete copywriting ruleset that governs tone, readability, SEO, GEO, and expertise signals. Every heading, paragraph, CTA, meta description, and FAQ answer must follow those rules.

Also read `docs/KEYWORD_MAP.md` for per-page keyword targets and `docs/CONTENT_BRIEF.md` for page-by-page content structure.

### Copy Quality Gates
Before committing any copy to a component, verify:
1. **Readability** — Flesch-Kincaid grade level 8-10 (accessible to business owners, not dumbed down)
2. **SEO** — Primary keyword in H1, first 100 words, and meta title. Secondary keywords in H2s.
3. **GEO** — At least one extractable definition block per practice area page. FAQ section with natural-language questions. All claims have specificity (numbers, years, case types).
4. **Expertise** — Every practice area page must reference specific legal mechanisms, statutes, or processes that only a knowledgeable firm would mention. No generic "we fight for you" filler.

## Component Patterns

### Section Layout
Never build a section that is just centered text above centered cards. Every section should have an intentional layout:
- Text-left / image-right (or reversed)
- Full-bleed image with text overlay on one side
- Narrow text column with a wide sidebar element
- Staggered cards with varying heights

### Attorney Cards
Each attorney card links to a dedicated bio page at `/attorneys/[slug]`. Display: professional headshot, name, title, direct phone, email. On the bio page: full credentials, bar admissions, education, practice focus, notable experience.

### Practice Area Pages
Each practice area gets its own page at `/practice-areas/[slug]`. Structure:
1. Hero with practice area name + one-sentence value proposition
2. "What we handle" section with specific claim types
3. "Our approach" section showing process/methodology
4. FAQ section (minimum 5 questions) — these are critical for GEO
5. CTA band with consultation offer
6. Related practice areas (internal links for SEO)

### Contact / Lead Capture
- Contact form on dedicated `/contact` page with fields: name, company, phone, email, practice area (dropdown), brief description
- Sticky CTA in header: "Schedule a Consultation"
- CTA band appears on every page above the footer
- Phone number always visible in header

## Structured Data (Schema.org)
Every page must include appropriate JSON-LD:
- **All pages**: `LegalService` organization schema with name, address, phone, practice areas
- **Attorney pages**: `Person` schema with `jobTitle`, `worksFor`, `alumniOf`
- **Practice area pages**: `Service` schema + `FAQPage` schema
- **Blog posts**: `Article` schema with author, datePublished, dateModified
- **Contact page**: `ContactPoint` schema

---

## SEO Technical Requirements
- All images: descriptive alt text (not "image1.jpg")
- Canonical URLs on every page
- XML sitemap at `/sitemap.xml`
- `robots.txt` allowing all major crawlers including GPTBot, PerplexityBot, ClaudeBot
- Open Graph + Twitter Card meta on every page
- Page titles follow pattern: `[Page Topic] | Nixon Jach Hubbard | Dallas [Practice Area] Attorneys`
- Meta descriptions: 150-160 chars, include primary keyword and location
- Internal linking: every practice area page links to related areas and relevant attorney bios
